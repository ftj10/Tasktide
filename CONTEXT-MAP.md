This repo uses a multi-context layout. Each context has its own domain glossary.

| Context    | File                   | Scope                                      |
| ---------- | ---------------------- | ------------------------------------------ |
| `frontend` | `frontend/CONTEXT.md`  | React UI, task logic, i18n, offline sync   |
| `backend`  | `backend/CONTEXT.md`   | Express API, MongoDB models, auth, push notifications |
